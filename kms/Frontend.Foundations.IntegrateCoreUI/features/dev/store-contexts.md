# Store Contexts (`@integrate-core-ui/store`)

- **Category:** dev
- **Status:** active
- **Owners:** UI Platform Team

## Summary

React-context-based state library that provides tab stores for `TabbedResources`, `Toolbox`, and `Explorer`. The shared `createTabContext(contextId)` factory produces CRUD hooks over a `TabItem[]` list; domain-specific wrappers expose friendlier names (`addResource`, `addTool`).

## Behavior

- `IntegrateCoreUiContext` (alias `IcuContextProvider`) is a composite provider that wraps `TabbedResourcesProvider`, `ToolboxProvider`, and `ExplorerProvider` in one go.
- Each underlying provider is generated by `createTabContext('tabbed-resources' | 'toolbox' | 'explorer')`, returning `{Provider, useTabStore}` that operate on `TabItem[]` with full CRUD:
  - `addTab / removeTab / updateTab / activateTab / pinTab / unpinTab / setVisible / setContainerIndex`.
- Domain wrappers alias the hook names: `useTabbedResources() → { addResource, ... }`, `useToolbox() → { addTool, ... }`.
- Per-tab contexts (`ResourceContext`, `ToolContext`) let descendants of a tab's subtree read the active tab via `useResource()` / `useTool()`.
- Although the package declares `react-redux` / `@reduxjs/toolkit` as peer dependencies, the source currently uses only React context — the Redux declaration is legacy or anticipatory.
